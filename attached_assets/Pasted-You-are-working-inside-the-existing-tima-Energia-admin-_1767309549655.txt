3️⃣ BENCHMARKING LOGIC (V1 — PRAGMATIC + DEFENSIBLE)

You are NOT building a forecasting engine.

V1 objective:

Given incomplete data, produce a conservative positioning:

“You appear above market band”

“You appear within band”

“We need more data to be confident”

Benchmark matching:

Match using:

segment (SME / Industrial)

region/submarket (SE/CO, S, NE, N)

contract term assumption (12/24/36)

product type assumption (convencional vs incentivada)
If not provided → choose “Unknown” and reduce confidence.

Output fields (minimum):

client_estimated_price_r_per_mwh (from bill OCR or manual)

benchmark_lower / benchmark_upper (R$/MWh)

gap_percent vs upper

potential_savings_range (min/max annual) must be phrased as estimate

recommendedNextStep:

REQUEST_RFQ (if above band or near upper)

NEED_MORE_DATA (if no_data/low confidence)

WAIT (if clearly below/within)

Confidence model:

Set confidence based on:

OCR confidence score

number of bill months available

benchmark freshness (lastUpdated)

completeness of UC/distributor/submarket
Return:

confidenceLevel (LOW/MED/HIGH)

confidenceReasons array (e.g., “Only 1 bill month uploaded”, “Benchmark outdated 120 days”, etc.)
