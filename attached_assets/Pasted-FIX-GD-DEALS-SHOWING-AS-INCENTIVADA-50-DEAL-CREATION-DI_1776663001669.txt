FIX: GD DEALS SHOWING AS "INCENTIVADA 50" — DEAL CREATION + DISPLAY
Problem
When creating a GD deal, the deal card shows "Incentivada 50 SE_CO" instead of "GD". Two root causes:

The "Criar Novo Deal" modal is still setting the energyType field on the deals table to "Incentivada 50" when GD is selected — it should either not set energyType at all for GD deals, or set it to something like "gd_assinatura"
The deal card in the deals registry (DealRegistry.tsx) displays deal.energyType + deal.submarket under the company name — it should show the track type (GD or ACL) instead, since that's the primary product line

Fix 1 — Deal creation modal
Find the "Criar Novo Deal" modal (likely in client/src/components/deals/DealRegistry.tsx or wherever the + Novo Deal button triggers).
When "GD" is selected as the product line:

Do NOT set energyType to "Incentivada 50" or "Convencional"
Either leave energyType null, or set it to "gd_assinatura"
The GDL track creation should already be happening from the previous fix — verify it still works

When "ACL" is selected:

Keep the energy type sub-selector (Convencional, Incentivada 50%, Incentivada 100%) since ACL deals need this
Set energyType based on the sub-selection

The modal flow should be:
Step 1: Select product line → GD or ACL

If GD selected:
  - No energy type sub-selector needed
  - Submarket can still be selected
  - Creates deal + GDL track

If ACL selected:
  - Show energy type sub-selector (Convencional, Incentivada 50%, etc.)
  - Submarket selection
  - Creates deal + ACL track
