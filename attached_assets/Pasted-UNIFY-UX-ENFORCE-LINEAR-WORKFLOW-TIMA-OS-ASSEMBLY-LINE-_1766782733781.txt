UNIFY UX + ENFORCE LINEAR WORKFLOW (ÓTIMA OS “ASSEMBLY LINE”)

You are working inside an existing production admin portal for Ótima Energia, a Brazilian B2B energy brokerage.

The portal already includes:

RBAC auth (admin / ops / sales)

Leads & Clients CRM

Bill upload + OCR

ECOS

Deal OS state machine + compliance gating

RFO/RFQ system + supplier tracking

Proposal system

Commission OS (Usage / Playbooks / Reconciliation / Cases)

Ops dashboard + notifications

Audit Trail

Now we need to unify the operational model so the system behaves like a single linear assembly line, not “a bunch of tabs”.

Goal: Sales + Ops can run 100 deals with zero chaos because the UI enforces the correct sequence and makes the next action obvious.

This build is primarily:

UX structure

Workflow enforcement

Guardrails

“Next actions” guidance

Reduced cognitive load

0️⃣ NON-NEGOTIABLE PRINCIPLES

One system, one flow
Users should feel: Lead → Client → Dossier → Deal → RFQ → Quotes → Proposal → Contract → Supply → Revenue.

You cannot skip steps
If a prerequisite is missing, the system must:

block the action,

explain why,

deep-link to fix it.

Role-based simplicity
Sales sees what they need to close.
Ops sees what they need to execute.
Admin sees everything.

No duplicate data entry
Where possible, auto-propagate:
Lead → Client → Dossier → Deal → RFQ → Proposal.

2️⃣ SALES DASHBOARD (LOW FRICTION, HIGH CLARITY)

Create a new Sales Dashboard page (or tab) that shows:

KPIs

New leads (last 7 days)

Leads awaiting response

Clients awaiting dossier/bill upload

Deals waiting on client decision (proposal sent)

Sales Worklist (“Today”)

A table with rows that represent “next best action”:

Lead: “Call / WhatsApp now”

Client: “Request bill / UC”

Deal: “Send RFQ” (only if allowed) OR “Waiting on quotes”

Proposal: “Follow-up”

Each row must have:

entity type (Lead/Client/Deal/Proposal)

name

status

blocker reason (if blocked)

one-click action button

This must reduce Sales tab hopping.

You already have Ops Dashboard with clickable KPIs.
Now upgrade it into a true “work engine”:

No hunting.

4️⃣ LINEAR WORKFLOW ENFORCEMENT (“ASSEMBLY LINE”)

Create a single canonical checklist of prerequisites for key steps and enforce them system-wide.

A) Define prerequisites for these actions:

Create Deal from Client

Send RFQ/RFO to supplier(s)

Mark QUOTES_RECEIVED

Generate Proposal

Move to OFFER_SELECTED

Move to ONBOARDING_PENDING

Move to CONTRACT_SIGNED

Move to SUPPLY_LIVE

Start Revenue tracking (commission events + usage periods)

B) Global blocker engine (shared helper)

Implement a backend helper that returns:

isBlocked: boolean

blockers: [{ code, title, description, deepLink }]

So UI can render:

red blocker banner

“Fix now” deep links

C) Blocker UI pattern (standardized)

Whenever blocked:

show a clear banner at top

list blockers in order

each has a “Go fix” button

NEVER silent failure
