FIX: "RESOLVER" AND "GERAR ECOS" BUTTONS STILL NOT WORKING
What's happening
On the bill-first journey strip, the deal shows:

Fatura: Completo ✓
ECOS: Bloqueado ✗ — "Gerar ECOS" button underneath
Dossiê: Completo ✓
RFQ: Bloqueado ✗

Below the strip there's a pink warning bar: "ECOS não gerado — Gere o perfil ECOS a partir dos dados da fatura" with a "Resolver >" button.
Neither "Gerar ECOS" nor "Resolver" does anything when clicked.
How to diagnose
Open the browser console (F12 → Console tab) and click both buttons. Look for:

JavaScript errors
Network requests that fire (or don't fire)
Console logs from click handlers

Then find these buttons in the code:
Button 1: "Gerar ECOS" — this is in the bill-first journey strip at the top of the deal detail. Search client/src/components/deals/DealAssemblyTab.tsx for "Gerar ECOS" or "Generate ECOS" or "ecos/generate".
Button 2: "Resolver" — this is in the pink blocker warning bar. Search the same file or client/src/components/deals/DealDetail.tsx for "Resolver" or "resolve".
Most likely causes
Cause A: onClick handler is missing or empty. The button renders but has no onClick prop, or the onClick calls a function that does nothing.
Cause B: onClick tries to navigate but the navigation doesn't work. For example, it might try window.location.href = '...' or router.push('...') but the target doesn't exist or the tab switch logic is broken.
Cause C: onClick calls the ECOS generate API but swallows the error. The API might return { ok: false, blockers: [{ code: "NO_PRC" }] } because there's no PRC pricing data, and the error handling doesn't show anything to the user.
Fix — all three buttons must work
"Gerar ECOS" button must call the ECOS generation endpoint directly:
Find the button in DealAssemblyTab.tsx and make sure its onClick does this:
typescriptconst generateEcos = async () => {
  try {
    const res = await fetch(`/api/deals/${dealId}/ecos/generate`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-session-id': sessionId,
      },
    });
    const data = await res.json();
    
    if (data.ok === false && data.blockers) {
      // Show blockers to user
      const blockerMessages = data.blockers.map((b: any) => b.message || b.code).join('\n');
      toast({
        title: isPt ? 'ECOS bloqueado' : 'ECOS blocked',
        description: blockerMessages,
        variant: 'destructive',
      });
      return;
    }
    
    if (!res.ok) {
      toast({
        title: isPt ? 'Erro' : 'Error',
        description: data.error || 'Failed to generate ECOS',
        variant: 'destructive',
      });
      return;
    }
    
    // Success
    toast({
      title: isPt ? 'ECOS gerado' : 'ECOS generated',
      description: isPt ? 'Perfil ECOS criado com sucesso' : 'ECOS profile created successfully',
    });
    
    // Refresh the assembly data and ECOS data
    queryClient.invalidateQueries({ queryKey: [`/api/deals/${dealId}/assembly`] });
    queryClient.invalidateQueries({ queryKey: [`/api/deals/${dealId}/ecos/snapshots`] });
    queryClient.invalidateQueries({ queryKey: [`/api/deals/${dealId}/ecos`] });
  } catch (err: any) {
    toast({
      title: isPt ? 'Erro' : 'Error',
      description: err.message || 'Network error',
      variant: 'destructive',
    });
  }
};
Then make sure the button element actually calls it:
