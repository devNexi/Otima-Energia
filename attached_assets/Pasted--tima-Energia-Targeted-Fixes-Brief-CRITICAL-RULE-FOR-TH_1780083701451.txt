## FIX 1 — Navbar background (homepage only)

**Problem:** The navbar sits on a white/light background which is completely jarring against the dark hero and all dark sections below it.

**Fix:** Change the navbar background to `transparent` when the page is scrolled to the top (so it floats over the plasma hero). When the user scrolls down past 80px, transition to `background: rgba(9,8,30,0.92); backdrop-filter: blur(12px)`. This is a standard "transparent on hero, solid on scroll" navbar pattern.

**Exact implementation:**
```tsx
// In the navbar component, add scroll listener:
const [scrolled, setScrolled] = useState(false);
useEffect(() => {
  const onScroll = () => setScrolled(window.scrollY > 80);
  window.addEventListener('scroll', onScroll);
  return () => window.removeEventListener('scroll', onScroll);
}, []);

// Apply to navbar wrapper:
style={{
  background: scrolled ? 'rgba(9,8,30,0.92)' : 'transparent',
  backdropFilter: scrolled ? 'blur(12px)' : 'none',
  borderBottom: scrolled ? '1px solid rgba(158,63,253,0.12)' : 'none',
  transition: 'all 0.3s ease',
  position: 'fixed',
  top: 0,
  left: 0,
  right: 0,
  zIndex: 50,
}}
```

**Also:** Change all navbar text links from their current dark/purple colour to `rgba(255,255,255,0.75)` with hover `#ffffff`. The "ECOS™" link stays `#9e3ffd`. The "Solicitar Diagnóstico" button stays as-is (purple background, white text).

**Also:** Add `padding-top` equal to navbar height (approx 64px) to the hero section so content doesn't sit under the fixed navbar.
