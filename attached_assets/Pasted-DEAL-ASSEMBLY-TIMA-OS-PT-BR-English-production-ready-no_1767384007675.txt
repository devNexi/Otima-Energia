GOAL

Build Deal Assembly: a guided, linear “assembly line” workflow that makes it extremely hard for Ops/Sales to do the wrong thing, and very easy to do the right thing — end-to-end:

Deal Created → Dossiê Ready → Dossiê Locked → RFQ Sent → Quotes Received → Quote Selected → Uplift & Proposal → Onboarding Checklist → Contract Signed → Supply Live

This must:

Reduce cognitive load

Surface “next step” and blockers

Enforce gates (hard where required)

Keep audit-proof evidence

Work in demo mode with seeded data

NON-NEGOTIABLE UX PRINCIPLES

One screen tells the truth: what’s the next step + what’s blocking it.

No dense text; use short tooltips, banners, and checklists.

Assume Ops users have zero energy market knowledge.

Everything bilingual-ready (PT-BR first; English translation support).

2) HARD GATING RULES (ASSEMBLY ENFORCEMENT)

Deal Assembly must enforce the linear flow using existing gates. Add missing checks if not present.

Required gates

GATE A: Can’t send RFQ unless Dossiê is LOCKED

If user clicks “Enviar RFQ” and dossier not LOCKED:

Block action

Show “Travar Dossiê” step as required

Explain why briefly

GATE B: Can’t generate proposal unless

At least 2 eligible quotes OR 1 quote + explicit override note by admin

Baseline exists (bill-derived or manual baseline)

Selected “Recomendado” quote chosen

GATE C: Can’t transition to CONTRACT_SIGNED unless compliance checklist complete

Must include “authority/signer verification” (decision-maker proof)

Must include “proposal sent” evidence entry

Must include onboarding docs set marked complete

GATE D: Sales users

Sales should see the assembly but with limited actions:

They can view progress, generate proposal (if allowed by your policy), log comms

They cannot bypass Ops compliance gates
(If your current RBAC already handles this, just ensure consistent behavior here.)
