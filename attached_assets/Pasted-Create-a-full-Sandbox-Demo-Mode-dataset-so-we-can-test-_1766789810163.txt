Create a full “Sandbox / Demo Mode” dataset so we can test the Blind Auction + Quote Intake + SLA + Winner selection + Deal progression end-to-end with fake data.

1) Add “Demo Data Seeder” (admin-only)

Create an admin-only action in Settings (or Ops Dashboard) called:

“Seed Demo Data (Sandbox)”

When clicked:

Create a batch of fake records with obvious names so nobody confuses them with real data.

Everything should include a prefix like:
[DEMO] or TEST-

Example naming conventions:

Clients: TEST Client 01 - Padaria, TEST Client 02 - Hospital, TEST Client 03 - Shopping

Deals: TEST Deal 01 - 24m Fixed, etc.

Suppliers: TEST Supplier 01 - Email, TEST Supplier 02 - WhatsApp, TEST Supplier 03 - Portal

Contacts: test.supplier01@otimaenergia.com.br etc.

UC codes: TESTUC0001, TESTUC0002

CNPJ: use a clearly fake placeholder like 00.000.000/0001-00 (or generate valid-format dummy numbers, but still labelled TEST)

Important: Tag all seeded records with a boolean:

isDemo = true

Clients

Client A: DRAFT

Client B: READY

Client C: LOCKED

Deals

3 deals:

Deal 1 in DRAFT

Deal 2 in RFQ_SENT

Deal 3 in QUOTES_RECEIVED

Suppliers + Playbooks

6 suppliers with different RFQ requirements:

Email-only (requires attachment)

Email-only (requires specific subject format + fields)

WhatsApp-only (requires message template)

WhatsApp + Email fallback

Portal submission (URL field required)

“Manual phone call” (forces internal task only)

Each supplier must have:

Active playbook version

SLA target (24h / 48h / 72h mix)

Required RFQ fields (simulate painful variety)

RFO / Dispatch rows

Create at least one RFO linked to Deal 2

Create dispatch tracking rows across 5 suppliers

Some marked:

SENT

VIEWED

OVERDUE

Quotes
