Hard requirements
1) Canonical product types (3)

Store exactly one of:

CONVENCIONAL

INC_I50 (Incentivada 50%)

INC_I100 (Incentivada 100%)

PRCs may include only 1, 2, or all 3 products. Do not assume uniformity.

Product detection rules:

If text contains “convencional” → CONVENCIONAL

If text contains “incentivada” with “50” or “50%” or “i50” → INC_I50

If text contains “incentivada” with “100” or “100%” or “i100” → INC_I100

If it says only “incentivada” with no %:

do not guess silently

either infer from column header/context OR mark product=NULL and reject the row with reason AMBIGUOUS_PRODUCT

2) Canonical submarkets (4)

SE_CO

S

NE

N

If a PRC provides only the 4 region names (Sudeste/Sul/Nordeste/Norte), it must still map cleanly to the 4 canonical submarkets.

reject row with OUT_OF_RANGE
