Bug 2 — Consumption Unit Conversion (kWh vs MWh)
What's wrong: The dossier shows:

Consumo Anual: 30.290,00 MWh
Consumo Médio Mensal: 1.057,50 MWh
Demanda de Pico: 6,00 kW

This is a small studio with 6 kW demand and Group B (Baixa Tensão). It does NOT consume 30,290 MWh/year — that's a factory. The real values are 30,290 kWh/year = 30.29 MWh/year and 1,057.50 kWh/month = 1.06 MWh/month.
Root cause: When the dossier auto-populates from bill data, it copies the kWh values directly into the MWh fields without dividing by 1000. The bill parser returns consumption in kWh (e.g., totalEnergyKwh: 121224.6). The dossier fields annualConsumptionMWh and averageMonthlyMWh expect MWh.
Fix: Find every place where dossier consumption is populated from bill data. Search server/routes.ts for where annualConsumptionMWh or averageMonthlyMWh is set from bill extraction data.
The conversion must be:
typescript// Bill data is in kWh — dossier stores MWh
const billEnergyKwh = parseFloat(String(bill.totalEnergyKwh || 0));
const billEnergyMwh = billEnergyKwh / 1000; // Convert kWh → MWh

// If we have multiple bills, average the monthly consumption
const avgMonthlyMwh = billEnergyMwh; // Each bill is typically one month
const annualMwh = avgMonthlyMwh * 12; // Annualize

// Store in dossier
dossier.averageMonthlyMWh = avgMonthlyMwh;
dossier.annualConsumptionMWh = annualMwh;
Also fix the auto-populate logic that runs when a dossier is first created from bill data. Search for:

Bug 3 — Eligibility Not Recalculating
What's wrong: Even with 30,290 in the consumption field (whether kWh or MWh), eligibility shows "Não elegível". If the value were truly 30,290 MWh, this client should be ACL_DIRECT. If the corrected value is 30.29 MWh, they should be GD_ELIGIBLE (on a GD track).
Root cause: Eligibility is not recalculated when consumption is edited. It was set once at creation and never updated.
Fix: Make eligibility auto-recalculate on EVERY dossier save.
In the dossier update endpoint (PATCH/PUT), add this after the fields are saved:
typescript// After saving dossier fields, recalculate eligibility
const annualMwh = parseFloat(String(updatedDossier.annualConsumptionMWh || 0));

// Get the deal's track type to make eligibility track-aware
let trackType: string | null = null;
const clientDeals = await storage.getDealsForClient(clientId);
if (clientDeals.length > 0) {
  const { dealTracks } = await import('@shared/schema');
  const tracks = await db.select().from(dealTracks).where(eq(dealTracks.dealId, clientDeals[0].id));
  trackType = tracks[0]?.type || null;
}

// Calculate eligibility based on track type
let newEligibility: string;
if (trackType === 'GDL') {
  newEligibility = annualMwh >= 6 ? 'GD_ELIGIBLE' : 'NOT_ELIGIBLE_YET';
} else if (trackType === 'ACL') {
  if (annualMwh >= 500) newEligibility = 'ACL_DIRECT';
  else if (annualMwh >= 50) newEligibility = 'ACL_VAREJISTA';
  else newEligibility = 'NOT_ELIGIBLE_YET';
} else {
  // No track — general thresholds
  if (annualMwh >= 500) newEligibility = 'ACL_DIRECT';
  else if (annualMwh >= 50) newEligibility = 'ACL_VAREJISTA';
  else if (annualMwh >= 6) newEligibility = 'GD_ELIGIBLE';
  else newEligibility = 'NOT_ELIGIBLE_YET';
}

// Update eligibility
await db.update(clientDossiers)
  .set({ eligibilityType: newEligibility, updatedAt: new Date() })
  .where(eq(clientDossiers.clientId, clientId));
