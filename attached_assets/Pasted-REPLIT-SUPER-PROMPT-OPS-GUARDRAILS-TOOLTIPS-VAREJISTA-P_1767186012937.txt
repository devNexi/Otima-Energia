3️⃣ OPS ERROR PLAYBOOKS (“IF X → DO Y”) — IN-PORTAL
Purpose

Remove decision paralysis. Every failure mode becomes a repeatable protocol.

Where it lives

Help → “Ops Playbook” tab

Inline alert appears when trigger condition happens

REQUIRED ERROR SCENARIOS

❌ Varejista não respondeu
Trigger:

RFQ sent + no response after SLA from playbook
Actions:

send reminder using preferred channel (email/WhatsApp)

log communication

escalate to named contact

dispatch to backup varejista(s)

❌ Cotação incoerente
Trigger:

price deviates from benchmarks by > X% OR missing fields
Actions:

verify dossier locked

verify product type + submarket

request clarification using required format

attach original file

log note

❌ Cliente mudou dado depois do RFQ
Trigger:

dossier changed post RFQ_SENT
Actions:

pause deal

notify all varejistas involved

decide: restart RFQ or proceed

document decision

❌ Faltando documento
Trigger:

onboarding cannot move because docs missing
Actions:

open checklist missing docs list

request docs via portal + WhatsApp

log contact + set due date

❌ Comissão divergente
Trigger:

reconciliation variance
Actions:

verify usage source + period

verify playbook snapshot

check supplier report format

open dispute + SLA

❌ Deal travado tempo demais
Trigger:

deal idle > X days
Actions:

identify blocker

create/assign case

set SLA

notify admin if overdue

5️⃣ SUPPLIER PLAYBOOK UPGRADE (REMOVE “VAREJISTA? Y/N” — ADD REAL DIFFERENTIATORS)
IMPORTANT

Your internal language can treat the whole 126 list as “varejistas”.
So do NOT store “Varejista: Y/N”.

Instead store operational differentiators that make them distinct.

Required new Supplier Playbook fields

Add these fields to supplierPlaybooks (or create supplierPlaybookOperational JSONB if you prefer):

A) Onboarding SLA + Response Behavior

onboarding_sla_days (number)

quote_response_sla_hours (number)

preferred_contact_channel (EMAIL / WHATSAPP / PORTAL)

relationship_notes (free text: “Falar com Maria, responde depois das 15h”)

B) Products Quoted

products_supported (array):

CONVENCIONAL

INCENTIVADA_50

INCENTIVADA_100

OTHER (free text)

C) Submarkets Covered

submarkets_supported (array):

SE_CO

S

NE

N

D) RFQ Intake Format

rfq_intake_method (EMAIL / PORTAL / WHATSAPP / OTHER)

rfq_template_required (boolean)

rfq_template_id (optional link to template)

rfq_required_fields (checklist array)

rfq_attachment_requirements (array: bills, UC list, contrato atual, etc)

E) Required Docs / Guarantees / Credit Turnaround

docs_required (array)

guarantees_supported (array: none / deposito / fiança / seguro garantia / outras)

credit_turnaround_days (number)

common_credit_reject_reasons (array)

F) Commission Reporting

commission_report_format (PDF / EXCEL / CSV / PORTAL / EMAIL_BODY)

commission_report_frequency (MONTHLY / QUARTERLY / UPFRONT / MIXED)

commission_report_fields_expected (array)

G) Consumption Variance Treatment

variance_method (TRUE_UP / CLAWBACK / OFFSET / OTHER)

variance_frequency (MONTHLY / QUARTERLY / ANNUAL)

variance_notes (free text: “faz compensação no mês seguinte”)
