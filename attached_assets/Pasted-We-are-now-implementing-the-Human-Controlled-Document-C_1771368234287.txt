-----------------------------------
PART 1 — SEND INTAKE LINK WORKFLOW
-----------------------------------

Inside the Track Intake modal:

1. Add contact verification block:
   - Editable contact name
   - Editable email (required, validated)
   - Editable WhatsApp number (E.164 format, +55…)
   - WhatsApp opt-in checkbox (required for WhatsApp sending)

2. Ensure Generate Link exists (already built).

3. Add a primary button:
   "SEND INTAKE LINK"

When clicked:

A. Validate:
   - Email format valid
   - If WhatsApp sending enabled: WhatsApp number valid + opt-in true

B. Execute:
   - Send email using existing SMTP
   - Log portal_access_logs entry: INTAKE_SENT
   - Store sentAt timestamp
   - Create Zoho task:
       Subject: "Follow up intake link – <Company>"
       Due: next business hour (America/Sao_Paulo)
   - Trigger webhook:
       POST /webhook/chase/start
       Include full payload (dealId, trackId, contact, link, expiry, requiredBills)

C. Ensure idempotency:
   - Multiple clicks must not send duplicates
   - Use unique eventId per track

-----------------------------------
PART 2 — PORTAL CHASE STATE ENGINE
-----------------------------------

Create new table: track_chase_state

Fields:
- trackId (FK)
- active (boolean)
- level (int 0–7)
- lastSentAt (timestamp)
- nextSendAt (timestamp)
- stoppedReason (nullable string)
- optOut (boolean)
- optOutAt (timestamp)
- createdAt
- updatedAt

Portal responsibilities:
- Start state when SEND pressed
- Stop state when:
    intakeCompleted
    manualOverride
    optOut
- Expose GET /api/tracks/:id/chase-state

Do NOT implement scheduling logic here.
Scheduling lives in n8n.
Portal only stores state.
