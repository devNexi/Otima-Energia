We already generate an “ECOS™ Insight Pack” PDF for a deal snapshot, but it currently reads like an internal debug printout and is not persuasive. Example current output (Deal ID e6ca5d58-53c8-4a03-93ba-7cde00b65805, Snapshot v1 ID #1) shows: company + CNPJ, consumption block with zeros, PRC min/max/avg and a talk track, but it’s text-heavy, unclear, and not visually compelling. It also exposes precise numeric benchmark values that we do not want to show directly to clients.

What’s wrong with the current ECOS PDF (must fix)

Not a sales document: It reads like a compliance report (“DADOS INSUFICIENTES”, “Confiança: BAIXA”, raw fields).

Too literal / too many raw numbers: It prints exact PRC min/max/avg and row detail lines (e.g. R$ 349.21 – R$ 405.61/MWh) and even shows “Rows: 2”. We want to keep benchmark numbers vague and show them visually.

No clear “where you stand” summary: The document does not clearly tell the client if they’re well above / above / on par / below / well below the market.

No persuasive narrative: Talk track is generic, not tailored to the client position vs market.

Design is bland: Needs to look kick ass, modern, on-brand, with Ótima Energia logo, brand fonts/colors, spacing, hierarchy, visualizations.

Goal

Turn ECOS into a client-facing sales tool that:

Communicates position vs market clearly and visually

Provides smart, valuable insight (not just “data”)

Includes tailored next-step guidance based on where they sit

Avoids sharing exact benchmark averages; show bands and relative position

Still supports ops/debug via an internal view, but the client PDF is polished

Market band boundaries (don’t show values)

Define band thresholds using PRC distribution:

Let marketMid = PRC average (or median)

Let marketSpread = (max-min)

Define bands by % distance from mid:

Well below: client <= mid * 0.90

Below: 0.90–0.97

On par: 0.97–1.03

Above: 1.03–1.10

Well above: > 1.10
(These thresholds can be tuned, but implement something deterministic.)
