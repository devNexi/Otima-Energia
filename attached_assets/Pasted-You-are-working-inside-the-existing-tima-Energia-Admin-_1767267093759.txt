Context

Goal

Ship a complete Portal Dictionary v1 that:

Displays real terms used in the portal

1) DATA MODEL (DB) — REQUIRED

Create a database table:

portal_dictionary_terms

id (uuid / serial)

term_key (string, unique) // internal key (snake_case)

term_pt (string) // PT-BR display title

term_en (string) // EN display title

definition_pt (text) // PT-BR definition (short, clear)

definition_en (text) // EN definition (short, clear)

category (enum/string) // one of: DOSSIER, RFQ_QUOTES, ONBOARDING, COMMISSION, OPERATIONS, GENERAL

aliases (text[]) // searchable synonyms e.g. ["UC", "Unidade Consumidora"]

related_terms (text[]) // list of term_keys (optional)

created_at

updated_at

Seed this table with the copy below (Section 4).

2) API — REQUIRED

Create endpoints:

GET /api/portal-dictionary

Query params:

q = search string (optional)

category = filter (optional, omit means All)

Behavior:

Full-text search across: term_pt, term_en, definition_pt, definition_en, aliases

Return ordered results: exact matches first, then partial matches

POST /api/portal-dictionary (admin only)

Create new term. Validate unique term_key.

PATCH /api/portal-dictionary/:id (admin only)

Update term.

DELETE /api/portal-dictionary/:id (admin only)

Soft delete preferred (optional), otherwise hard delete OK.

RBAC:

Everyone logged in can read

Only admin can create/edit/delete

Audit:

Log create/edit/delete to audit trail with source="portal_dictionary"
