1. CORE DATABASE SCHEMA
If no suppliers table exists, create it with these fields:

sql
TABLE suppliers:
- id (uuid, primary key)
- created_at
- updated_at
- name (text, not null) -- e.g., "Thymos Energia"
- alias (text, nullable) -- e.g., "thymos"
- is_active (boolean, default true)
- contact_name (text)
- contact_email (text)
- contact_phone (text)
- notes (text)
- metadata_json (jsonb) -- for future flexibility
If no interaction logging exists, create this table:

sql
TABLE supplier_interactions:
- id (uuid, primary key)
- created_at
- supplier_id (foreign key to suppliers.id)
- interaction_type (text) -- ENUM: 'RFQ_SENT', 'QUOTE_RECEIVED', 'CALL', 'EMAIL', 'MEETING', 'NOTE'
- interaction_source (text) -- e.g., 'quote_inbox', 'manual_entry', 'zoho_crm'
- deal_id (foreign key to deals.id, nullable)
- rfq_id (foreign key to rfqs.id, nullable)
- note (text) -- For manual notes or automated context (e.g., "Quote received via email for Deal #123")
- metadata_json (jsonb) -- e.g., {'quote_value': 50000, 'response_time_hours': 48}

3. AUTOMATED LOGGING & KPI CALCULATION
A. Auto-Log Interactions (CRITICAL)
The system must automatically create supplier_interactions records for:

RFQ Sent: When an RFQ is dispatched via the system, log RFQ_SENT. Link the rfq_id and deal_id.

Quote Received: When a quote is confirmed via the Quote Inbox (Priority 1), log QUOTE_RECEIVED. Link the deal_id. Store the response_time_hours in metadata_json (calculated from RFQ sent time to quote received time).

Manual Logging: On the supplier detail page, provide a quick form for Renan to log a CALL, EMAIL, MEETING, or NOTE.

B. Auto-Calculate KPIs
Calculate and display the following for each supplier:

Average Response Time (h): The mean response_time_hours from all QUOTE_RECEIVED interactions where that data is available.

Quote Win Rate (%): (Number of deals won where supplier's quote was selected) / (Number of RFQs sent to this supplier) * 100.

Requires linking final deal.supplier_awarded_id to the supplier table.

4. FOLLOW-UP DASHBOARD
On the main /admin/suppliers page, implement a separate, highlighted section at the top called "Ações Pendentes".

This section lists two types of suppliers:

Sem Contato > 7 Dias: Suppliers with is_active = true and no supplier_interactions in the last 7 days.

Cotações Atrasadas: Suppliers with an RFQ_SENT log in the last 30 days, but no subsequent QUOTE_RECEIVED log, and where the linked RFQ's deadline has passed.

Each listed supplier should have a "Registrar Contato" button that opens a quick log form.
