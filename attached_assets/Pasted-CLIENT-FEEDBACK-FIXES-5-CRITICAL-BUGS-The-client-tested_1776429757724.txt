BUG 3 — GD ELIGIBILITY NOT RECOGNIZED IN DOSSIER
What's wrong: The dossier shows "Não elegível / Em análise" for a client with 10.29 MWh annual consumption. The eligibility logic only checks ACL thresholds (≥500 MWh = ACL Direct, ≥50 MWh = ACL Varejista). But for GD, the threshold is much lower — GD BT requires only ≥500 kWh/month (~6 MWh/year).
Current broken logic (in server/routes.ts and server/deal-ecos-engine.ts):
typescriptif (annualConsumptionMwh >= 500) eligibility = 'ACL_DIRECT';
else if (annualConsumptionMwh >= 50) eligibility = 'ACL_VAREJISTA';
else eligibility = 'NOT_ELIGIBLE_YET';
How to fix:
The eligibility calculation must consider the deal's track type (GD vs ACL). Update the logic:
typescriptfunction calculateEligibility(annualConsumptionMwh: number, trackType?: string): string {
  // GD eligibility — much lower threshold
  if (trackType === 'GDL' || trackType === 'GD') {
    // GD BT: >= 500 kWh/month = ~6 MWh/year
    if (annualConsumptionMwh >= 6) return 'GD_ELIGIBLE';
    // GD MT: higher consumption, Subgrupo A4
    if (annualConsumptionMwh >= 50) return 'GD_MT_ELIGIBLE';
    return 'GD_UNDER_REVIEW';
  }

  // ACL eligibility
  if (annualConsumptionMwh >= 500) return 'ACL_DIRECT';
  if (annualConsumptionMwh >= 50) return 'ACL_VAREJISTA';

  // Below ACL but may qualify for GD
  if (annualConsumptionMwh >= 6) return 'GD_ELIGIBLE_ONLY';

  return 'NOT_ELIGIBLE_YET';
}
Apply this fix in ALL places where eligibility is calculated:

server/routes.ts — search for eligibility = 'ACL_DIRECT' (appears around lines 3977-3980, 4160-4162, 10166-10168)
server/deal-ecos-engine.ts — if eligibility is calculated there
client/src/components/dossier/ClientDossierTab.tsx — if eligibility display logic is client-side

Also update the dossier display to show the correct eligibility label in Portuguese:
